Guide for Estimating AI Assistance Percentage

Estimation Methods

1. Time-Based Method

Basic calculation:

AI Assistance % = (Time with AI / Total project time) × 100

Example:

  • Total project time: 20 hours
  • Time using AI: 4 hours
  • Estimation: 20% AI assistance

2. Content/Code Lines Method

For code:

AI Assistance % = (AI-generated lines / Total lines) × 100

For documentation:

AI Assistance % = (AI-generated words / Total words) × 100

Example:

  • Total code: 500 lines
  • AI-generated code: 150 lines
  • Estimation: 30% AI assistance

3. Project Components Method

Component Weight AI Assistance Contribution
Data analysis 30% 10% 3%
SQL code 25% 60% 15%
Visualizations 20% 20% 4%
Documentation 15% 40% 6%
Presentation 10% 80% 8%
TOTAL 100% - 36%

Assistance Levels by Activity

Level 1: Minimal Assistance (0-20%)

  • Information lookup: Syntax verification, documentation
  • Debugging: Simple error identification
  • Formatting: Style correction, indentation

Examples:

  • "How do you write a function in Python?"
  • "What's the correct syntax for a JOIN in SQL?"

Level 2: Moderate Assistance (21-50%)

  • Base code generation: Basic structures, templates
  • Guided analysis: Methodological approach suggestions
  • Content review: Improvements to existing documentation

Examples:

  • Generate function skeleton
  • Suggest appropriate libraries for analysis
  • Review and improve code comments

Level 3: Significant Assistance (51-80%)

  • Algorithm development: Complete implementation with human review
  • Data analysis: Statistical analysis script generation
  • Content creation: Complete section writing

Examples:

  • Implement complete clustering algorithm
  • Generate exploratory data analysis
  • Create complex visualizations

Level 4: Extensive Assistance (81-100%)

  • Automatic generation: Content generated almost entirely by AI
  • Complete implementation: End-to-end solutions
  • Automatic writing: Auto-generated technical documentation

Examples:

  • Complete AI-generated dashboard
  • Complete technical documentation
  • Automatically generated presentation

Self-Assessment Framework

Guiding Questions for Students

1. Initiative and Conceptualization (Who defined WHAT to do?)

  • I completely defined the problem and approach (0 - 25%)
  • AI helped me refine my initial idea (26 - 50%)
  • AI suggested the main approach (51 - 75%)
  • AI completely defined the solution (76 - 100%)

2. Implementation (Who did the technical work?)

  • I wrote all code/content from scratch (0 - 25%)
  • AI gave me structure/templates that I completed (26 - 50%)
  • AI generated code that I significantly modified (51 - 75%)
  • AI generated most content with minimal modification (76 - 100%)

3. Understanding and Validation (Do you understand what you did?)

  • I understand and can explain every line/concept (0 - 25%)
  • I understand general concepts but not all details (26 - 50%)
  • I understand the result but not the complete process (51 - 75%)
  • I don't fully understand how it works (76 - 100%)

4. Problem Solving (Who solved the errors?)

  • I identified and resolved all problems (0 - 25%)
  • AI helped me identify problems that I solved (26 - 50%)
  • AI solved problems that I identified (51 - 75%)
  • AI identified and solved most problems (76 - 100%)

Formula:

Self-Assessment Score = (Q1 + Q2 + Q3 + Q4) / 4

AI Usage Log Template

For each work session:

## Session [Date] **Duration:** [X hours] **Objectives:** [What you wanted to accomplish] ### AI Usage: - **Tool:** [Claude, ChatGPT, etc.] - **Time with AI:** [X minutes/hours] - **Prompts used:** [Approximate number] - **Type of assistance:** - [ ] Code generation - [ ] Debugging - [ ] Concept explanation - [ ] Content review - [ ] Data analysis - [ ] Documentation ### Results: - **Generated by AI:** [Description] - **Modified by me:** [What changes you made] - **Created by me:** [What you did without AI] ### Assistance estimation for this session: [X%]

BEST PRACTICE: Commit regularly and document session details in commit messages

Suggested Final Calculation

Combined Formula:

Final AI Assistance % = (0.25 × Time %) + (0.35 × Content %) + (0.25 × Complexity %) + (0.15 × Self-Assessment Score)

Where:

  • Time %: Proportion of time using AI
  • Content %: Proportion of content generated by AI
  • Complexity %: Complexity level of tasks performed by AI
  • Self-Assessment Score: Based on the Self-Assessment Framework

Practical Examples

Example: Database Project

Activity: Relational database design Time % = (6 / 20) × 100 = 30% Time Contribution = 0.25 × 30% = 7.5% Content % = (200 / 500) × 100 = 40% Content Contribution = 0.35 × 40% = 14% Complexity Score = 62.5% (Significant level) Complexity Contribution = 0.25 × 62.5% = 15.625% Self-Assessment Score = (25 + 75 + 25 + 25) / 4 = 37.5% Self-Assessment Contribution = 0.15 × 37.5% = 5.625% Final AI Assistance % = 7.5% + 14% + 15.625% + 5.625% = 42.75%

Recommendations for Professors

1. Establish Acceptable Ranges

  • Learning projects: 0-30% recommended
  • Intermediate projects: 31-60% acceptable
  • Advanced projects: 61-80% with justification
  • Research projects: Variable according to objectives

2. Require Evidence

  • Git control version or similar
  • AI conversation history
  • Detailed time log
  • Explanation of modifications made

3. Complementary Assessment

  • Oral exams about the project
  • Specific technical questions
  • Request live modifications
  • Peer review among students

Red Flags

Indicators of excessive AI dependence:

  • Student cannot explain basic concepts in their project
  • Code/content has inconsistent style or approach
  • Student struggles to make simple modifications
  • Documentation is too polished relative to code quality
  • Student cannot debug simple errors independently

Best Practices for Students

DO:

  • Keep detailed logs of AI interactions
  • Always test and verify AI-generated code
  • Add personal insights and analysis
  • Modify AI content to match your style
  • Ask follow-up questions to understand concepts

DON'T:

  • Copy-paste AI output without understanding
  • Use AI for concepts you should learn manually
  • Hide or minimize AI usage
  • Submit AI content as completely original work
  • Rely on AI for fundamental course concepts
